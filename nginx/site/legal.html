User agreement and privacy policy

These pages describe MatrixAPI account use, OpenAI-compatible API access, balance top-up, token handling, and privacy practices. They stay inside the MatrixAPI domain and open in the same tab.

User agreement Privacy policy

User agreement

Use MatrixAPI for lawful API access and account management.

Account responsibility

You are responsible for the account, API tokens, request activity, and balance usage under your account.

  • Keep passwords, tokens, and client import links private.
  • Rotate a token immediately if it may have been exposed.
  • Use accurate contact information for billing and support.

Acceptable use

MatrixAPI provides routed access to AI models through compatible endpoints and dashboard tools.

  • Do not abuse rate limits, payment flows, or upstream providers.
  • Do not use the service for unlawful, harmful, or unauthorized activity.
  • Respect model, provider, and regional restrictions shown in the dashboard.

Privacy policy

MatrixAPI collects only the data needed to operate the gateway.

Data we process

MatrixAPI may process account identifiers, balance records, request metadata, model names, token usage, and operational logs.

  • API secrets are sensitive and should never be shared in support messages.
  • Billing records are used to reconcile Alipay orders and account balance.
  • Logs are used for debugging, abuse prevention, and usage visibility.

Data protection

Access is limited to operational needs. Sensitive credentials should be rotated after exposure.

  • Use HTTPS endpoints and OpenAI-compatible clients with secure storage.
  • Review token activity and disable unused tokens in the dashboard.
  • Contact support if you need account or billing data reviewed.

Billing terms

Alipay top-up is balance-based.

MatrixAPI currently exposes Alipay online top-up. A recharge quantity of 100 corresponds to an actual payment of 100 CNY when the configured price is 1.0 CNY per quantity unit.

Support

Contact MatrixAPI without sharing full credentials.

For account, token, billing, or provider questions, contact support by email. Include route names, timestamps, and masked token prefixes only.
