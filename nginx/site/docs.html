02 / Overview

Overview

MatrixAPI routes compatible requests to the models enabled for your account and records usage against the selected token.

Use the model name shown in the MatrixAPI model plaza. Token permissions, balance, and provider availability are checked before a request is forwarded.
03 / Quick start

Quick start

The shortest path to a successful request is: sign in, create a token, then send an OpenAI-compatible chat call.

1. Sign inOpen the MatrixAPI console and confirm that your wallet has a positive balance.
2. Set the base URLUse https://matrixapi.online/v1 in your SDK or client provider settings.
3. Send a requestPass the token as a Bearer credential and use an enabled model identifier.
04 / Node.js

Node.js

Install the official OpenAI package and point its base URL at MatrixAPI.

Download Node.js

npm install openai

import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.MATRIX_API_KEY,
  baseURL: "https://matrixapi.online/v1",
});

const completion = await client.chat.completions.create({
  model: "gpt-5.4",
  messages: [{ role: "user", content: "Hello from MatrixAPI" }],
});

console.log(completion.choices[0]?.message?.content);
05 / Endpoints

Endpoints

Use the same MatrixAPI host for all OpenAI-compatible requests.

https://matrixapi.online/v1
GET /modelsList models available to the current token.
POST /chat/completionsSend a chat completion with OpenAI message format.
POST /images/generationsRequest an image when the selected model supports image generation.
curl https://matrixapi.online/v1/chat/completions -H "Authorization: Bearer $MATRIX_API_KEY" -H "Content-Type: application/json" -d '{"model":"gpt-5.4","messages":[{"role":"user","content":"Hello MatrixAPI"}]}'

08 / Codex++

Codex++

Configure Codex++ with the OpenAI-compatible provider profile and MatrixAPI token.

OPENAI_BASE_URL=https://matrixapi.online/v1
OPENAI_API_KEY=YOUR_MATRIX_API_TOKEN
OPENAI_MODEL=gpt-5.4
If Codex++ offers a provider selector, choose OpenAI-compatible so the custom base URL is retained.

10 / Claude

Claude

Use the OpenAI-compatible mode in a Claude client that supports custom providers. Set MatrixAPI as the provider base URL and select a model visible in Model Plaza.

Do not mix an Anthropic-native endpoint with the MatrixAPI OpenAI-compatible path. The supported MatrixAPI route is https://matrixapi.online/v1.

12 / Billing

Billing and top-up

MatrixAPI uses balance-based billing. Online top-up currently exposes Alipay only; successful payment callbacks credit the wallet automatically.

Review current model pricing and token usage in the console. Keep enough balance for the request type and selected model.
Open walletView model plaza
13 / Errors

Errors and troubleshooting

401 / 403Check that the Bearer token is valid, enabled, and copied without surrounding quotes.
402 / insufficient balanceOpen Wallet and top up before retrying the request.
404 / model not foundUse the exact model identifier returned by GET /models.
429 / rate limitSlow retries and respect the limits shown for your account or provider.
400 / invalid requestValidate JSON, message roles, and the endpoint path after /v1.
TimeoutConfirm the client uses HTTPS and the /v1 base URL, then retry once.
When contacting MatrixAPI support, include the route, timestamp, status code, and a masked token prefix. Never send the full token or payment credentials.
