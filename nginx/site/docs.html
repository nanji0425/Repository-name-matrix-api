3 分钟接入 Matrix API

Matrix API 提供 OpenAI 兼容接口。你只需要创建 API Key，把 Base URL 设置为 https://matrixapi.online/v1，即可在常见客户端、SDK 或自己的应用中调用模型。

01 / 快速开始

快速开始

第一次使用建议按“注册账号 → 充值余额 → 创建 API Key → 发送测试请求”的顺序完成。

01注册并登录使用账号密码注册 Matrix API，登录后进入控制台。
立即注册
02充值余额打开钱包页，选择充值金额并完成支付。支付成功后余额会自动入账。
打开钱包
03创建 API Key在控制台创建专用 API Key，并妥善保存到客户端或服务端配置中。
创建 Key
02 / API 接入

API 接入

Matrix API 使用 OpenAI 兼容格式。大多数支持自定义 OpenAI Base URL 的 SDK 和工具，都可以直接填写下面的地址。

https://matrixapi.online/v1
GET /models查看当前 API Key 可用的模型列表。
POST /chat/completions使用 OpenAI messages 格式发起对话请求。
Authorization请求头填写 Bearer YOUR_API_KEY。
curl https://matrixapi.online/v1/chat/completions \
  -H "Authorization: Bearer $MATRIX_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-5.4","messages":[{"role":"user","content":"你好，Matrix API"}]}'

Node.js 示例

npm install openai

import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.MATRIX_API_KEY,
  baseURL: "https://matrixapi.online/v1",
});

const completion = await client.chat.completions.create({
  model: "gpt-5.4",
  messages: [{ role: "user", content: "你好，Matrix API" }],
});

console.log(completion.choices[0]?.message?.content);
03 / 客户端配置

客户端配置

所有客户端都使用同一套填写方式：服务商类型选择 OpenAI 兼容，Base URL 填 https://matrixapi.online/v1，API Key 填 Matrix API 控制台创建的 Key。

Cherry Studio设置 → 模型服务 → 添加 OpenAI 兼容服务，填写 Base URL 与 API Key 后测试连接。
Chatbox模型提供方选择自定义 OpenAI，填入接口地址和 API Key，再选择模型。
LobeChat添加 OpenAI 兼容服务，接口地址使用 Matrix API Base URL。
NextChat在自定义接口配置中填写 Base URL 和 API Key，模型名使用模型广场显示的名称。
OpenCat / DeepChat选择 OpenAI 兼容或自定义服务商，完成后发送一句测试消息。
Cursor / Continue在扩展配置里使用 OpenAI Compatible Provider，并把 Key 放在本地安全配置中。
如果客户端要求填写完整接口地址，请填写 https://matrixapi.online/v1；如果它会自动拼接 /v1，则按客户端说明填写。
04 / 模型与价格

模型与价格

模型广场展示的价格为 Matrix API 当前对外价格。实际费用会根据所选模型、输入 Token、输出 Token、任务类型和计费单位自动计算。

模型名称调用接口时请使用模型广场显示的完整模型名。
输入 / 输出价格长上下文、推理、视觉等能力不同，费用也可能不同。
如何核对每次请求的模型、Token 和消耗可在使用日志中查看。
查看模型广场
05 / 充值与余额

充值与余额

Matrix API 使用余额计费。当前在线充值支持支付宝，支付完成后系统会根据支付回调自动入账。

01打开钱包进入钱包页查看当前余额和可选充值金额。
02确认付款点击确认付款后会打开支付页面，请在新页面完成支付。
03检查到账支付成功后返回控制台查看余额；如未到账，请联系支持并提供订单时间。
打开钱包
06 / 使用日志

使用日志

如果你想核对某次调用的费用或排查错误，请先查看控制台使用日志。

时间与状态确认请求发生时间、是否成功、是否超时或被拒绝。
模型与 Token查看模型名、输入 Token、输出 Token 和请求类型。
消耗金额每次请求的实际消耗会记录在日志中，便于核对余额变化。
查看使用日志
07 / 常见问题

常见问题

401 / 403检查 API Key 是否正确、是否启用，复制时不要带空格或引号。
余额不足请先充值余额，再重新发起请求。
模型不存在使用模型广场或 GET /models 返回的完整模型名。
请求超时检查网络、客户端超时设置和模型响应时间，必要时稍后重试。
参数错误确认 JSON 格式、message role、endpoint 路径和模型能力是否匹配。
支付未到账保留支付时间和订单信息，联系 Matrix API 支持核对。
08 / 账户安全

账户安全

API Key 等同于调用凭证，请像密码一样保管。

不要公开 Key不要把完整 API Key 发到截图、群聊、代码仓库或前端页面里。
按用途创建不同客户端或项目建议使用不同 API Key，方便单独停用。
泄露后处理立即停用或删除旧 Key，创建新 Key 并更新客户端配置。

联系支持
