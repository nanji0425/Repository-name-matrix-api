# MatrixAPI 文档、品牌与管理员后台修复设计

## 目标

在现有 `new-api + Nginx 静态注入` 生产架构上，完成四项用户可见改进：

1. 所有页面 Logo 在当前尺寸基础上缩小 25%，保持透明背景与白底 favicon。
2. `/docs` 成为完整的 MatrixAPI 站内文档，不再把 `docx.kkkliao.cn` 作为跳转目标；参考站只用于内容结构与教程顺序。
3. 修复管理员后台的渠道检测失败反馈、状态刷新和模型描述编辑/展示链路，并覆盖其他后台页面的错误状态。
4. 用自动化回归覆盖公开页面、登录后台、模型同步/搜索、充值弹窗、余额门禁、文档锚点、移动端布局和生产运行状态。

## 现状与根因线索

- `nginx/site/brand-init.js` 当前对首页、认证页、导航 Logo 使用 96/80/48/72px 注入尺寸，需要统一按 0.75 缩放，并复核 `public-shell.css` 和原生控制台 CSS。
- 静态首页、排行榜和文档页仍包含 `https://docx.kkkliao.cn/` 外链，数据库初始化脚本和 `brand-init.js` 也把该地址作为 `docs_link`；这些值必须改为站内 `/docs`。
- 原生模型编辑表单已经有 `description` 字段和 payload 转换，需重点验证接口保存、列表/详情展示和空值回填，避免重复增加字段。
- 渠道检测前端通过 `/api/channel/test/:id` 调用 `handleTestChannel`，请求配置跳过通用错误处理；需要统一解析 `success/message/upstream_status/error`，在批量和单条检测后刷新缓存，并保留可操作的失败原因。

## 设计

### 1. 品牌尺寸

保留现有透明 PNG/SVG 资源和白底 favicon，仅调整实际显示尺寸：

| 场景 | 当前 | 目标 |
| --- | ---: | ---: |
| 首页主 Logo | 96px | 72px |
| 登录/注册 Logo | 80px | 60px |
| 公共导航 Logo | 96px | 72px |
| 控制台顶部 Logo | 48px | 36px |
| 控制台侧栏 Logo | 72px | 54px |

尺寸由同一组 CSS 变量/注入规则控制，避免静态页与 React 页面漂移；移动端只允许在不溢出的前提下按同一比例缩小。

### 2. 站内文档

`/docs` 保持静态、无需登录、同标签页展示。页面包含：购买后流程、文档说明、快速开始、Node.js 下载、接口地址、CC Switch、Codex++、Codex、Claude Code、VS Code、常见报错。章节导航全部使用本页 `#anchor`，站点首页、排行榜、模型广场、控制台顶部导航统一链接 `/docs`。

内容适配 MatrixAPI：

- 主接口为 `https://matrixapi.online/v1`，Claude 兼容路径说明使用 MatrixAPI 域名，不复制参考站的域名、兑换码或联系方式。
- 只保留站内登录、API Key、钱包、模型广场入口；外部下载地址仅在确有下载用途时保留，不能作为“文档”导航。
- `DocsLink`、`general_setting.docs_link`、运行时 `/api/status` 的 `docs_link` 统一为 `/docs`。

### 3. 管理员后台

- 渠道检测：将单渠道检测、批量检测和模型检测统一为可取消的 mutation；成功时更新 `response_time/test_time` 并刷新渠道查询，失败时显示 HTTP 状态、上游消息和建议动作，不把空响应误报成成功。
- 模型描述：保留现有 `description` 字段，补齐表单标签/占位、保存后缓存刷新、表格单元格查看和详情回填；空描述显示明确的“未设置”，不吞掉接口错误。
- 后台页面：渠道、模型、部署、用户、兑换码、订阅、系统设置和个人资料均加入固定等待、404/空白页、浏览器 console error 和关键文本检查。

### 4. 验收与回滚

- 本地先执行静态脚本、前端构建、Go 测试和 QA；通过后备份服务器并重建当前镜像。
- 生产只读检查包括容器健康、Nginx `-t`、HTTPS、安全头、公开路由、`/api/status`、模型列表和 Redis/数据库状态。
- 不创建真实支付订单；充值只打开支付宝确认弹窗。
- 任一关键回归失败时停止发布，保留现有备份并恢复上一镜像，不修改 `.env`、证书、支付密钥或上游 Key。

## 成功标准

- 线上所有页面 Logo 比当前版本缩小 25%，透明背景正常，无移动端溢出。
- 页面和 `/api/status` 不再把 `docx.kkkliao.cn` 作为文档跳转目标；点击文档只留在 `matrixapi.online/docs`。
- 管理员渠道检测能够显示成功/失败的真实原因并刷新状态；模型描述可新增、编辑、保存、查看和回填。
- 公开路由、安全、运行时、后台、模型广场、充值、游乐场和移动端回归均无失败项。
